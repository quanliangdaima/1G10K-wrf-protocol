# Contributing to WRF

Thank you for your interest in contributing to the WRF protocol and its ecosystem.

This repository contains the open WRF protocol specification, reference templates, and documentation. It does **not** contain the WRF Deck application source code.

## What we welcome

- New WRF protocol templates in `templates/`
- Documentation improvements and translations in `docs/`
- Third-party tool adapters and integration examples
- Bug reports and feature requests related to the protocol specification, templates, or documentation via GitHub issues

## Security issues

**Do not open a public issue or pull request for security vulnerabilities.**

See [`SECURITY.md`](SECURITY.md) and email **contact@1g10k.dev** instead.

## What requires discussion first

- Changes to the core WRF protocol specification (`docs/SPEC.md`)
- Breaking changes to YAML keys, block structure, or file format
- Modifications to the reference template `templates/wrf/execution-track.wrf.md`

Please open a GitHub issue before submitting a pull request for any of the above. The core protocol specification is maintained by the 1G10K team so that compatibility and upgrade paths stay consistent across the ecosystem.

## Design discussions

For open-ended questions or design discussions about the WRF protocol itself — such as new YAML keys, block structure, or ecosystem interoperability — please use [GitHub Discussions](https://github.com/1G10K/wrf-protocol/discussions).

For security issues, private commercial matters, or anything involving undisclosed product plans, email **contact@1g10k.dev** instead of opening a public discussion.

## How to contribute

1. Fork this protocol repository.
2. Create a feature branch for your change.
3. Update relevant documentation, examples, and the template checksum if needed.
4. Open a pull request with a clear description of the change and why it improves the protocol or its documentation.

## Community standards

Be respectful, constructive, and assume good intent. We are building an open protocol for safe human-AI collaboration.
