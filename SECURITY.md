# Security Policy

## Supported Versions

- WRF Protocol **v3.1** is currently supported for security updates. Versions **v3.0 and earlier** are no longer maintained and do not receive security patches.

## Reporting a Vulnerability

If you discover a security vulnerability in the WRF Protocol, WRF Deck, or related tools, please:

- Email **contact@1g10k.dev**
- Do not disclose vulnerability details in public issues, PRs, forums, or social media.

We will acknowledge receipt within 5 business days and publicly credit you upon release of the fix, if you wish.

## Information to Include

To help us process your report efficiently, please include:

- Affected version
- Vulnerability type and impact
- Reproduction steps and a minimal PoC
- Relevant WRF excerpt or code path
- Suggested fix (optional)

## Disclosure Policy

This project follows the principles of Coordinated Vulnerability Disclosure:

1. After receiving a report, we will acknowledge it and assign a handler.
2. We will validate the impact and prepare a fix; details will not be made public until the fix is released.
3. Once the fix is released, we will disclose it via GitHub Security Advisories / Release Notes and credit the reporter.
4. Critical fixes will be notified via contact@1g10k.dev.

## Incident Response

If you suspect the WRF Protocol or WRF Deck has been exploited:

1. Stop executing the relevant command immediately.
2. Click **Lock Sentry Evidence** to lock the current WRF file state.
3. Report via **contact@1g10k.dev**, providing WRF excerpts, reproduction steps, and environment details.

## Security Updates

Security patches and announcements will be released through:

- GitHub Security Advisories
- Repository Release Notes
- Email notifications (if subscribed)

## Security Team

WRF security matters are handled by the 1G10K core maintainers. All reports will be kept confidential until coordinated disclosure is complete.
